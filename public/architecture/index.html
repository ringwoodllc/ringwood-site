System architecture

How the Ringwood field platform is built, where the data lives, what it is written in, and how everything connects. Written so it can be shared with a partner, not kept in code or in someone's head.

The big picture

Everything runs through one program on Cloudflare. The same program serves the website and the field apps, checks who is signed in, and talks to the database, the file storage, and the AI on the server side so secrets never reach the phone. The data lives in a Postgres database (Supabase). Photos live in Supabase file storage. The AI is Anthropic's Claude.

What it is written in

Three things, each suited to its job. There is no heavy framework and no build step for the app pages, which keeps it fast and easy to change.

PartLanguageWhere it runsWhat it does
The engine (back end)JavaScriptCloudflare WorkerOne file, src/index.js. Routing, sign-in, the API, and all the server-side calls to the database, storage, and AI.
The app screens (front end)HTML + CSS + JavaScriptThe phone / browserThe pages you tap through. A progressive web app (installable, works offline at the shell level). No framework, no build step.
The dataSQL (PostgreSQL)SupabaseThe information itself, kept in tables (see the schema below). The Worker reads and writes it over an automatic REST API, so it sends JSON rather than raw SQL.
The iOS wrapperCapacitoriPhoneWraps the same web app so it can ship through the App Store / TestFlight and reach the camera.

In short: the database is SQL tables. A table is like a spreadsheet with named columns and one row per record. The Worker (JavaScript) is the only thing that touches the tables, and it does so with its secret key, so the browser never holds a password to the data.

The schema in detail

Each table, column by column. uuid is a unique id. text is words. text[] is a list of words (for example a list of photo URLs). timestamptz is a date and time. jsonb is a small structured blob. PK is the primary key (the unique id of a row). FK points at another table.

How a request flows

  1. You tap something in the app on your phone.
  2. The phone sends an HTTPS request to the Cloudflare Worker (and only the Worker).
  3. The Worker checks your signed-in cookie and what you are allowed to see.
  4. The Worker reads or writes the Postgres tables, uploads or fetches photos from Storage, or asks Claude to read a photo or draft text. Your secrets stay on the Worker.
  5. The Worker sends back just the JSON the screen needs, and the screen updates.
